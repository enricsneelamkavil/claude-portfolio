How I work

Process, briefly.

  1. 01

    Listen

    Spend a week in the product, the docs and on calls. Understand the team before the screens.

  2. 02

    Frame

    Write down the one sentence the work has to be true to. Everything checks back against it.

  3. 03

    Sketch wide

    Paper first, then low-fi. Many cheap options before any single expensive one.

  4. 04

    Make it real

    High-fi, prototypes, motion. Build with engineering, not at them.

  5. 05

    Finish

    Sweat the boring details. Empty states, errors, loading, pause states. It's all the product.
